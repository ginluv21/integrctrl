# integrctrl - полный разбор проекта

Вариант 2.5. Утилита командной строки на языке Си для Linux.
Назначение: контроль целостности файлов через MD5-хеши.

## Как работает программа в целом

Программа запускается с флагами командной строки:

```
integrctrl -s [-r] -f <файл_базы> <директория>   - сохранить хеши
integrctrl -c [-r] -f <файл_базы> [директория]   - проверить целостность
```

**Режим -s (save):**
1. Обходит указанную директорию, для каждого файла считает MD5-хеш.
2. Все найденные файлы и папки записывает в бинарный файл базы.

**Режим -c (check):**
1. Читает базу из файла.
2. Снова обходит директорию (или ту, что была записана в базе).
3. Сравнивает старые и новые данные, выводит статус каждого объекта.

## Структуры данных

### file_rec (include/integrctrl.h)

```c
typedef struct file_rec {
    int id;               // уникальный номер записи (начинается с 1)
    char name[256];       // имя файла или папки (только имя, не полный путь)
    int type;             // TYPE_DIR=0 или TYPE_FILE=1
    int parent_id;        // id родительской папки; у корневой записи parent_id=0
    unsigned char md5[16]; // 16 байт MD5-хеша; для папок заполнено нулями
} file_rec;
```

Интересный момент - полный путь нигде не хранится. Хранится только имя и ссылка
на родителя. Чтобы получить полный путь, нужно пройти по цепочке parent_id вверх.
Это экономит место и позволяет легко переименовывать папки в будущем.

Корневая запись - это запись с parent_id=0. Её имя - это путь к директории,
которую сканировали (например "/home/user/project"). Все остальные записи имеют
parent_id, указывающий на их родителя.

### prog_opts (include/integrctrl.h)

```c
typedef struct {
    int mode;         // 's' или 'c' (char хранится как int)
    int recursive;    // 1 если был флаг -r, иначе 0
    char *db_file;    // указатель на строку из argv[] с именем файла базы
    char *target_dir; // указатель на строку из argv[] с именем директории
} prog_opts;
```

Указатели db_file и target_dir НЕ копируют строку - они просто указывают на
элементы массива argv[]. Это нормально, потому что argv[] существует всё время
работы программы.

### db_header (include/db.h)

```c
typedef struct {
    unsigned int magic;        // 0x41634463 = "AcDc" в ASCII
    unsigned int record_count; // сколько file_rec записано в файле
    int recursive;             // был ли флаг -r при сохранении
    time_t saved_at;           // время сохранения (секунды с 01.01.1970)
} db_header;
```

Это заголовок бинарного файла базы. Он записывается первым, перед всеми
file_rec. Magic number нужен чтобы убедиться, что файл действительно является
базой integrctrl, а не каким-то другим файлом.

time_t - это тип из <time.h>, обычно 8 байт (int64_t). Хранит количество секунд
прошедших с 1 января 1970 года ("эпоха Unix"). Функция time(NULL) возвращает
текущее время в этом формате.

### vector_t (include/vector.h)

```c
typedef struct {
    file_rec **data;   // массив указателей на file_rec
    file_rec **res;    // резервный буфер для быстрого расширения
    size_t len;        // текущее количество элементов
    size_t cap;        // текущая ёмкость (сколько можно хранить без расширения)
    int owns_data;     // 1 = free каждый элемент при уничтожении, 0 = не трогать
} vector_t;
```

data - это массив УКАЗАТЕЛЕЙ, то есть каждый элемент вектора - это указатель
на file_rec, выделенный отдельно через malloc(). Сам вектор не хранит
file_rec напрямую, только адреса.

Поле res (резерв) - это второй буфер заранее выделенной памяти. Когда основной
буфер data заканчивается, вектор переключается на res (он уже выделен, не надо
ждать malloc) и одновременно выделяет новый res. Это делает расширение быстрее.

## Стандартные функции, которые используются в программе

Здесь разобраны все стандартные функции Си и системные вызовы, которые встречаются
в коде. Понимание каждой из них - это основа понимания всей программы.

### malloc / free (stdlib.h)

```c
void *malloc(size_t size);
void free(void *ptr);
```

`malloc(n)` просит у операционной системы n байт оперативной памяти и возвращает
указатель на начало выделенного блока. Если памяти не хватило - возвращает NULL.

Важно: выделенная память не инициализируется - в ней может быть любой мусор.
Поэтому после malloc нужно либо заполнить поля вручную, либо использовать memset.

`free(ptr)` возвращает блок памяти операционной системе. После free пользоваться
указателем ptr нельзя - там уже чужие данные или вообще недоступная память.

В программе malloc используется везде где создаётся file_rec:
```c
file_rec *rec = malloc(sizeof(file_rec));
if (rec == NULL) {
    // обработка ошибки - malloc вернул NULL
}
```
sizeof(file_rec) вычисляется компилятором и равен 284 байтам.

### realloc (stdlib.h)

```c
void *realloc(void *ptr, size_t new_size);
```

Изменяет размер уже выделенного блока памяти. Может переместить блок в другое
место если нужно его расширить - поэтому результат ВСЕГДА нужно сохранять в
переменную и использовать её, а не старый указатель.

В vec_resize:
```c
file_rec **temp = realloc(vec->data, new_cap * sizeof(file_rec *));
if (temp == NULL)
    return;  // ошибка, старый массив не тронут
vec->data = temp;
```
Если записать `vec->data = realloc(vec->data, ...)` и realloc вернёт NULL,
то мы потеряем указатель на старый массив - утечка памяти. Поэтому сначала
сохраняем в temp, проверяем, и только потом присваиваем.

### memset (string.h)

```c
void *memset(void *ptr, int value, size_t count);
```

Заполняет count байт начиная с адреса ptr значением value.
value передаётся как int, но используется как unsigned char (0..255).

В программе используется для обнуления MD5-хеша у папок:
```c
memset(rec->md5, 0, 16);
```
Это заполняет все 16 байт нулями. Нули означают "хеш не вычислен".

### memcpy (string.h)

```c
void *memcpy(void *dst, const void *src, size_t count);
```

Копирует count байт из src в dst. Буферы не должны пересекаться.
В vec_copy используется для копирования file_rec целиком:
```c
memcpy(new_rec, *(vec->data + i), sizeof(file_rec));
```
Копирует 284 байт - все поля структуры разом, включая массив md5[16].

### strcmp / strncmp (string.h)

```c
int strcmp(const char *s1, const char *s2);
int strncmp(const char *s1, const char *s2, size_t n);
```

Сравнивают строки символ за символом. Возвращают:
- 0 если строки одинаковы
- отрицательное если s1 < s2 (лексикографически)
- положительное если s1 > s2

strcmp сравнивает до нулевого терминатора '\0'.
strncmp сравнивает не более n символов - безопаснее если строки могут быть
не завершены нулём.

В check_integrity:
```c
if (strcmp(old_path, prefix) == 0 ||
    (strncmp(old_path, prefix, plen) == 0 && old_path[plen] == '/'))
```
Первая часть: путь полностью совпадает с prefix (это сама папка-фильтр).
Вторая часть: путь начинается с prefix + '/' (это файл внутри папки-фильтра).

### strcpy / strncpy (string.h)

```c
char *strcpy(char *dst, const char *src);
char *strncpy(char *dst, const char *src, size_t n);
```

strcpy копирует строку включая '\0'. Опасна если dst мал.
strncpy копирует не более n символов. Если src длиннее n - '\0' не добавляется!
Поэтому всегда вручную добавляем '\0' после strncpy:

```c
strncpy(rec->name, entry->d_name, NAME_LEN - 1);
rec->name[NAME_LEN - 1] = '\0';  // гарантируем завершение строки
```

NAME_LEN = 256. Записываем не более 255 символов, 256-й байт всегда '\0'.

### strncat (string.h)

```c
char *strncat(char *dst, const char *src, size_t n);
```

Дописывает не более n символов из src в конец dst (находит '\0' в dst и пишет туда).
Всегда добавляет '\0' в конец. В build_path:
```c
strncat(buf, rec->name, size - len - 2);
```
size - len - 2: оставляем место чтобы точно не выйти за границу буфера.

### strlen (string.h)

```c
size_t strlen(const char *s);
```

Возвращает длину строки - количество символов до '\0', не включая сам '\0'.
В build_path используется чтобы найти конец уже написанного пути:
```c
size_t len = strlen(buf);
buf[len] = '/';
buf[len + 1] = '\0';
```

### strchr / strrchr (string.h)

```c
char *strchr(const char *s, int c);   // первое вхождение символа c
char *strrchr(const char *s, int c);  // последнее вхождение символа c
```

Возвращают указатель на найденный символ или NULL если не нашли.

strchr в check_integrity - проверяем есть ли '/' в пути (значит путь вложенный):
```c
if (!recursive && strchr(search, '/') != NULL)
    continue;  // вложенный файл, пропускаем
```

strrchr в run_save - находим последний '/' в пути к базе:
```c
char *slash = strrchr(dir_buf, '/');
if (slash != NULL) {
    *slash = '\0';  // обрезаем имя файла, оставляем только папку
    mkdir(dir_buf, 0755);
}
```
Например из "data/db/base.db" получаем "data/db" и создаём эту папку.

### snprintf (stdio.h)

```c
int snprintf(char *buf, size_t size, const char *format, ...);
```

Форматирует строку как printf, но записывает не более size-1 символов в buf,
и всегда добавляет '\0'. Безопаснее sprintf, потому что не переполняет буфер.
Возвращает количество символов которое было бы записано без ограничения.

```c
snprintf(full_path, sizeof(full_path), "%s/%s", dir_path, entry->d_name);
```
Собирает полный путь: "src" + "/" + "main.c" = "src/main.c".

### fopen / fclose (stdio.h)

```c
FILE *fopen(const char *path, const char *mode);
int fclose(FILE *f);
```

fopen открывает файл и возвращает FILE* - непрозрачный указатель на структуру
буфера. Возвращает NULL при ошибке (файл не существует, нет прав и т.д.).

Режимы:
- "r" - чтение текста
- "w" - запись текста (создаёт или очищает)
- "rb" - чтение бинарного файла
- "wb" - запись бинарного файла

'b' (binary) - на Linux не меняет поведение, на Windows отключает
автозамену '\n' ↔ '\r\n'. Используем из принципа - код должен работать везде.

fclose закрывает файл и сбрасывает буфер на диск. Всегда вызываем после работы,
иначе данные могут не записаться (они хранятся в буфере в памяти).

### fwrite / fread (stdio.h)

```c
size_t fwrite(const void *ptr, size_t size, size_t count, FILE *f);
size_t fread(void *ptr, size_t size, size_t count, FILE *f);
```

fwrite записывает count элементов размером size байт из памяти ptr в файл f.
fread читает count элементов размером size байт из файла f в память ptr.
Обе возвращают количество успешно записанных/прочитанных элементов.

Ключевой момент: они работают с сырыми байтами, без какого-либо преобразования.
Структура записывается ровно такой, какой лежит в памяти.

```c
fwrite(&hdr, sizeof(db_header), 1, f);
```
Читать: "запиши 1 элемент размером sizeof(db_header) байт из адреса &hdr в файл f".

```c
if (fread(&hdr, sizeof(db_header), 1, f) != 1) {
    // не прочитали ни одного элемента - файл пуст или повреждён
}
```

### opendir / readdir / closedir (dirent.h)

```c
DIR *opendir(const char *path);
struct dirent *readdir(DIR *dir);
int closedir(DIR *dir);
```

Три функции для чтения содержимого директории.

opendir открывает директорию и возвращает DIR* - дескриптор директории.
Возвращает NULL если директория не существует или нет прав.

readdir читает следующий элемент. Возвращает указатель на статичный struct dirent
(не нужно free!) или NULL когда элементы закончились.

struct dirent - минимально нужные поля:
```c
struct dirent {
    char d_name[256];  // имя файла или папки
    unsigned char d_type; // тип: DT_REG, DT_DIR, DT_LNK, ...
};
```

DT_REG = обычный файл (REGular file)
DT_DIR = директория (DIRectory)
DT_LNK = символическая ссылка (LiNK) - пропускаем

closedir закрывает дескриптор. Аналог fclose для директорий.

### getopt (unistd.h)

```c
int getopt(int argc, char *argv[], const char *optstring);
extern char *optarg;   // аргумент текущего флага
extern int optind;     // индекс следующего необработанного элемента argv
```

Стандартная функция разбора флагов командной строки.
Вызывается в цикле while, каждый вызов обрабатывает один флаг.
Возвращает символ найденного флага или -1 когда флаги закончились.

Строка "scrf:" расшифровывается:
- каждый символ = один флаг
- двоеточие после символа = флаг принимает аргумент (optarg будет указывать на него)

Если встречен неизвестный флаг - getopt печатает ошибку и возвращает '?'.

После цикла optind указывает на первый не-флаговый аргумент. Именно так
получаем путь к директории (он идёт после всех флагов).

### mkdir (sys/stat.h)

```c
int mkdir(const char *path, mode_t mode);
```

Создаёт директорию. mode = права доступа в восьмеричном формате.
0755 = владелец: полный доступ (7=rwx), группа и остальные: чтение+запуск (5=r-x).

Возвращает 0 при успехе, -1 при ошибке (папка уже есть - тоже ошибка EEXIST).
В программе ошибку mkdir игнорируем намеренно - если папка уже есть, это нормально.

### time (time.h)

```c
time_t time(time_t *t);
```

Возвращает текущее время в секундах с 1 января 1970 00:00:00 UTC.
Если передать NULL - просто возвращает время, не записывает никуда.

```c
hdr.saved_at = time(NULL);
```

### localtime / strftime (time.h)

```c
struct tm *localtime(const time_t *timer);
size_t strftime(char *buf, size_t max, const char *format, const struct tm *tm);
```

localtime переводит time_t (число секунд) в структуру struct tm с полями:
tm_year (лет с 1900), tm_mon (месяц 0..11), tm_mday (день), tm_hour, tm_min, tm_sec.

strftime форматирует tm в строку по шаблону. Коды формата:
- %Y - год 4 цифры (2025)
- %m - месяц 01..12
- %d - день 01..31
- %H - часы 00..23
- %M - минуты 00..59
- %S - секунды 00..59

```c
struct tm *tm_info = localtime(&saved_at);
strftime(time_buf, sizeof(time_buf), "%Y-%m-%d %H:%M:%S", tm_info);
// результат: "2025-05-25 14:32:01"
```

## Полный листинг src/main.c с разбором

```c
#include "integrctrl.h"   // подключаем наш заголовочный файл со структурами и прототипами

int main(int argc, char *argv[]) {
    // argc - количество аргументов командной строки (включая имя программы)
    // argv - массив строк: argv[0]="./integrctrl", argv[1]="-s", argv[2]="-f", ...

    prog_opts opts;
    // объявляем структуру opts на стеке (не через malloc)
    // поля не инициализированы - parse_args заполнит их все

    parse_args(argc, argv, &opts);
    // передаём &opts (адрес структуры) чтобы parse_args могла её заполнить
    // если аргументы неверные - parse_args вызовет exit() и мы сюда не вернёмся

    if (opts.mode == 's')
        run_save(&opts);   // режим сохранения хешей
    else
        run_check(&opts);  // режим проверки целостности
    // opts.mode гарантированно 's' или 'c' после parse_args

    return 0;  // 0 = успешное завершение программы
}
```

## Полный листинг src/integrctrl.c с разбором

### parse_args

```c
void parse_args(int argc, char *argv[], prog_opts *opts) {
    int opt;  // будет хранить символ текущего флага (-1 когда флаги кончились)

    // инициализируем все поля opts перед разбором
    opts->mode = -1;          // -1 = режим ещё не задан
    opts->recursive = 0;      // по умолчанию не рекурсивно
    opts->db_file = NULL;     // NULL = файл базы ещё не задан
    opts->target_dir = NULL;  // NULL = директория ещё не задана

    while ((opt = getopt(argc, argv, "scrf:")) != -1) {
        // getopt возвращает символ флага или -1 когда флаги кончились
        // "scrf:" - допустимые флаги: s, c, r, f (f требует аргумент)
        switch (opt) {
        case 's':
            opts->mode = 's';  // запоминаем режим save
            break;
        case 'c':
            opts->mode = 'c';  // запоминаем режим check
            break;
        case 'r':
            opts->recursive = 1;  // включаем рекурсию
            break;
        case 'f':
            opts->db_file = optarg;
            // optarg - глобальная переменная getopt, указывает на строку-аргумент
            // например для "-f data/base.db" optarg = "data/base.db"
            break;
        default:
            // неизвестный флаг - getopt уже напечатал сообщение об ошибке
            print_usage(argv[0]);  // печатаем справку и выходим
        }
    }

    if (optind < argc) {
        opts->target_dir = argv[optind];
        // optind - индекс первого не-флагового аргумента
        // например для "./integrctrl -s -f db src/" это будет "src/"
    }

    // проверяем что обязательные параметры заданы
    if (opts->mode == -1 || opts->db_file == NULL) {
        fprintf(stderr, "Ошибка: пропущены обязательные параметры\n");
        print_usage(argv[0]);
        // print_usage вызывает exit() - дальше не идём
    }

    if (opts->mode == 's' && opts->target_dir == NULL) {

        fprintf(stderr, "Ошибка: для режима -s нужно указать директорию\n");
        print_usage(argv[0]);
    }
```

### scan_directory

```c
void scan_directory(const char *dir_path, int parent_id, int *next_id,
                    vector_t *vec, int recursive) {
    DIR *dir = opendir(dir_path);
    // открываем директорию, получаем дескриптор
    if (dir == NULL) {
        // ошибка - папка не существует или нет прав на чтение
        fprintf(stderr, "Ошибка: не удалось открыть директорию %s\n", dir_path);
        return;  // выходим, дальше ничего не делаем
    }

    struct dirent *entry;
    // entry будет указывать на текущий элемент директории

    while ((entry = readdir(dir)) != NULL) {
        // читаем следующий элемент; NULL = элементы кончились
        // Читаем объекты внутри папки один за другим

        // Пропускаем служебные ссылки . и ..
        if (strcmp(entry->d_name, ".") == 0 ||
            strcmp(entry->d_name, "..") == 0) {
            // "." = ссылка на текущую папку, ".." = ссылка на родительскую
            // если не пропустить - уйдём в бесконечную рекурсию
            continue;  // переходим к следующему элементу
        }

        // Обрабатываем только папки и файлы
        if (entry->d_type != DT_DIR && entry->d_type != DT_REG) {
            // DT_LNK (символические ссылки), DT_FIFO (именованные каналы) и
            // другие типы пропускаем - нам они не нужны
            continue;
        }

        // Создаем file_rec
        file_rec *rec = malloc(sizeof(file_rec));
        // выделяем 284 байта на кучe (heap) для новой записи
        if (rec == NULL) {
            fprintf(stderr, "Ошибка выделения памяти для файла %s\n",
                    entry->d_name);
            continue;  // не смогли выделить память - пропускаем этот файл
        }

        rec->id = *next_id;  // Заполняем поля
        // разыменовываем указатель next_id чтобы получить текущее значение счётчика
        (*next_id)++;  // счетчик для некс файла
        // скобки обязательны: *next_id++ сначала разыменует, потом сдвинет указатель
        // а нам нужно увеличить значение по адресу, поэтому (*next_id)++

        rec->parent_id = parent_id;
        // id родительской папки передан как параметр

        strncpy(rec->name, entry->d_name, NAME_LEN - 1);
        // копируем имя файла в rec->name, не более NAME_LEN-1 = 255 символов
        rec->name[NAME_LEN - 1] = '\0';
        // гарантируем нулевой терминатор на последней позиции

        memset(rec->md5, 0, 16);
        // обнуляем хэш по умолчанию - для папок он останется нулями

        if (entry->d_type == DT_REG) {  // Обработка файлов и папок
            rec->type = TYPE_FILE;  // TYPE_FILE = 1

            char full_path[1024];
            snprintf(full_path, sizeof(full_path), "%s/%s", dir_path, entry->d_name);
            // собираем полный путь: "src" + "/" + "main.c" = "src/main.c"
            // нужен для fopen - opendir/readdir дают только имя без пути

            FILE *fp = fopen(full_path, "rb");
            // открываем файл для чтения в бинарном режиме
            if (fp != NULL) {
                md5File(fp, rec->md5);
                // вычисляем MD5-хеш файла, результат записывается в rec->md5 (16 байт)
                fclose(fp);  // обязательно закрываем
            } else {
                fprintf(stderr, "Ошибка чтения файла: %s\n", full_path);
                // хеш остаётся нулевым (memset выше), запись всё равно добавим
            }

        } else if (entry->d_type == DT_DIR) {
            rec->type = TYPE_DIR;  // TYPE_DIR = 0
            // хеш для папки = нули (уже сделано memset выше)
        }

        vec_push(vec, rec);
        // добавляем запись в вектор

        if (entry->d_type == DT_DIR && recursive) {
            // если это папка И нужна рекурсия - спускаемся вглубь
            char full_path[1024];
            snprintf(full_path, sizeof(full_path), "%s/%s", dir_path, entry->d_name);
            scan_directory(full_path, rec->id, next_id, vec, recursive);
            // рекурсивный вызов:
            // full_path = путь к подпапке
            // rec->id = parent_id для содержимого подпапки
            // next_id передаём по указателю - продолжаем нумерацию
        }
    }

    closedir(dir);  // закрываем дескриптор директории
}
```

```c
void run_check(prog_opts *opts) {
    printf("\n[Проверка] Начинаем сверку с базой %s...\n", opts->db_file);

    int db_recursive = 0;    // сюда запишем флаг рекурсии из базы

        return;
    }

    // Читаем папку из базы
    file_rec *db_root_rec = vec_get(old_vec, 0);
    // индекс 0 = первый элемент = корневая запись (записана первой в run_save)
    const char *db_root = db_root_rec->name;
    // имя корня = путь к директории которую сканировали при сохранении

    // передал папку иначе из базы
    const char *target_dir = (opts->target_dir != NULL) ? opts->target_dir : db_root;
    // тернарный оператор: если пользователь указал папку - берём её, иначе из базы
    // это позволяет проверить конкретную подпапку: integrctrl -c -f base.db src/utils

    // Вычисляем отн путь от db_root до target_dir
    char prefix[1024] = "";
    // prefix будет пустым если target_dir == db_root (проверяем всё)
    if (opts->target_dir != NULL && strcmp(opts->target_dir, db_root) != 0) {
        // пользователь указал другую папку - нужен prefix для фильтрации
        char root_buf[1024], tgt_buf[1024];
        strncpy(root_buf, db_root, sizeof(root_buf) - 1);
        strncpy(tgt_buf, opts->target_dir, sizeof(tgt_buf) - 1);

        // убираем trailing slash (финальный слеш)
        size_t rlen = strlen(root_buf);
        if (rlen > 0 && root_buf[rlen - 1] == '/') root_buf[rlen - 1] = '\0';
        size_t tlen = strlen(tgt_buf);
        if (tlen > 0 && tgt_buf[tlen - 1] == '/') tgt_buf[tlen - 1] = '\0';
        // убираем чтобы "src/" и "src" считались одним и тем же

        size_t root_len = strlen(root_buf);
        if (strncmp(tgt_buf, root_buf, root_len) == 0 && tgt_buf[root_len] == '/') {
            // target начинается с db_root + '/'
            // например db_root="src", target="src/utils" → prefix="utils"
            strncpy(prefix, tgt_buf + root_len + 1, sizeof(prefix) - 1);
        } else {
            // target - просто имя подпапки без полного пути
            char *sl = strrchr(tgt_buf, '/');
            strncpy(prefix, sl ? sl + 1 : tgt_buf, sizeof(prefix) - 1);
            // если есть '/' берём часть после него, иначе всю строку
        }
    }

    // форматируем дату сохранения базы
    char time_buf[64];
    struct tm *tm_info = localtime(&saved_at);
    // переводим time_t (секунды) в структуру с годом, месяцем, днём и т.д.
    strftime(time_buf, sizeof(time_buf), "%Y-%m-%d %H:%M:%S", tm_info);
    // форматируем в строку "2025-05-25 14:32:01"
    printf("[Проверка] База сохранена: %s\n", time_buf);
    printf("[Проверка] Директория: %s\n", target_dir);

    // создаём вектор для текущего состояния директории
    vector_t *new_vec = vec_create(10);
    int next_id = 1;

    file_rec *root_rec = malloc(sizeof(file_rec));
    root_rec->id = next_id++;
    root_rec->parent_id = 0;
    root_rec->type = TYPE_DIR;
    strncpy(root_rec->name, target_dir, NAME_LEN - 1);
    memset(root_rec->md5, 0, 16);
    vec_push(new_vec, root_rec);

    // Рекурсия берем из аргументов если не указана из базы
    int recursive = opts->recursive;
    // db_recursive читается из базы, но НАМЕРЕННО не используется
    // пользователь должен явно указать -r если хочет рекурсивную проверку

    scan_directory(target_dir, root_rec->id, &next_id, new_vec, recursive);

    printf("\n----------------------------------------\n");
    int stats[4] = {0, 0, 0, 0};
    // stats[0]=OK, stats[1]=CHANGED, stats[2]=MISSING, stats[3]=NEW
    // инициализируем нулями через {0, 0, 0, 0}

    check_integrity(old_vec, new_vec, prefix[0] ? prefix : NULL, stats, recursive);
    // prefix[0] ? prefix : NULL - если prefix не пустой передаём его, иначе NULL
    // prefix[0] = первый символ строки; если '\0' (пусто) - false, иначе true

    printf("----------------------------------------\n");
    printf("OK: %d  CHANGED: %d  MISSING: %d  NEW: %d\n",
           stats[0], stats[1], stats[2], stats[3]);
    printf("[Проверка] Объектов в базе: %zu, в текущем скане: %zu\n",

    vec_destroy(new_vec);  // освобождаем память текущего скана
```

```c
#include "check.h"

// Линейный поиск записи по id в векторе
static file_rec *find_by_id(vector_t *vec, int id) { // Ищем запись по id
    for (size_t i = 0; i < vec_len(vec); i++) {
        file_rec *rec = vec_get(vec, i);
        // vec_get возвращает указатель на элемент по индексу
        if (rec->id == id)
            return rec;  // нашли - возвращаем указатель (не копию!)
    }
    return NULL;  // не нашли
}

// Рекурсивно строит путь "папка/подпапка/файл" для заданной записи
static void build_path(vector_t *vec, file_rec *rec, char *buf, size_t size) { // Строим путь от корня до rec
    if (rec->parent_id == 0) {
        // это корневая запись - её имя не включаем в путь
        buf[0] = '\0';  // пустая строка
        return;
    }

    file_rec *parent = find_by_id(vec, rec->parent_id); // Ищем родителя
    if (parent == NULL || parent->parent_id == 0) {
        // родитель - корень (parent_id == 0) или не найден
        // значит rec - объект первого уровня, его путь = просто его имя
        strncpy(buf, rec->name, size - 1);
        buf[size - 1] = '\0';
        return;
    }

    build_path(vec, parent, buf, size); // Рекурсивно строим путь до родителя
    // после этого вызова в buf лежит путь к родителю, например "sub"

    size_t len = strlen(buf);  // длина уже написанного пути
    if (len + 1 < size) {
        buf[len] = '/';         // дописываем разделитель
        buf[len + 1] = '\0';    // временно завершаем строку
        strncat(buf, rec->name, size - len - 2);
        // дописываем имя текущего файла
        // size - len - 2: len уже занято, 1 под '/', 1 под '\0' = -2
    }
}

// Находит запись в векторе по относительному пути
static file_rec *find_by_path(vector_t *vec, const char *path) { // Ищем запись по пути от корня
    char buf[2048];
    for (size_t i = 0; i < vec_len(vec); i++) {
        file_rec *rec = vec_get(vec, i);
        if (rec->parent_id == 0)
            continue;  // пропускаем корневую запись (её путь пустой)
        build_path(vec, rec, buf, sizeof(buf));
        // строим путь для каждой записи и сравниваем
        if (strcmp(buf, path) == 0)
            return rec;  // нашли совпадение
    }
    return NULL;  // не нашли
}

// Побайтовое сравнение двух MD5-хешей
static int md5_equal(unsigned char *a, unsigned char *b) { // Сравниваем два MD5-хеша
    for (int i = 0; i < 16; i++)  // MD5 всегда 16 байт
        if (a[i] != b[i])
            return 0;  // нашли отличие - хеши разные
    return 1;  // все 16 байт совпали - хеши одинаковы
}

void check_integrity(vector_t *old_vec, vector_t *new_vec, const char *prefix,
                     int stats[4], int recursive) {
    char old_path[2048];   // буфер для пути из базы
    char new_path[2048];   // буфер для пути из текущего скана
    char search[4096];     // буфер для пути поиска (может быть prefix+path)

    int has_prefix = (prefix != NULL && prefix[0] != '\0');
    // has_prefix = 1 если нужна фильтрация по подпапке
    size_t plen = has_prefix ? strlen(prefix) : 0;
    // длина prefix - вычисляем один раз чтобы не считать в каждой итерации

    // === ЦИКЛ 1: ищем MISSING и CHANGED ===
    // проходим по всем записям из базы

    for (size_t i = 0; i < vec_len(old_vec); i++) {
        file_rec *old = vec_get(old_vec, i);
        if (old->parent_id == 0)
            continue;  // пропускаем корневую запись

        build_path(old_vec, old, old_path, sizeof(old_path));
        // строим путь этой записи в контексте old_vec

        if (has_prefix) {
            // фильтруем: оставляем только записи внутри подпапки prefix
            int match =
                strcmp(old_path, prefix) == 0 ||
                (strncmp(old_path, prefix, plen) == 0 && old_path[plen] == '/');
            // совпадение если: путь == prefix ИЛИ путь начинается с "prefix/"
            if (!match)
                continue;  // запись не в нашей подпапке - пропускаем
            if (strcmp(old_path, prefix) == 0)
                continue;  // сама папка prefix - её не выводим, только содержимое

            strncpy(search, old_path + plen + 1, sizeof(search) - 1);
            // отрезаем prefix + '/' из начала пути
            // "utils/file.c" → "file.c" если prefix="utils"
        } else {
            strncpy(search, old_path, sizeof(search) - 1);
            // без prefix - search = полный путь
        }

        if (!recursive && strchr(search, '/') != NULL)
            continue;
        // если не рекурсивно И в search есть '/' - это вложенный файл, пропускаем
        // strchr вернёт не-NULL если '/' найден в строке

        file_rec *cur = find_by_path(new_vec, search);
        // ищем соответствующую запись в текущем состоянии директории

        const char *t = (old->type == TYPE_FILE) ? "FILE" : "DIR ";
        // строка для вывода типа в таблице

        const char *status;
        const char *color;
        if (cur == NULL) {
            // запись есть в базе, но нет в текущем скане - файл удалён
            status = "MISSING";
            color = "\033[31m";  // красный
            stats[2]++;  // счётчик MISSING
        } else if (old->type == TYPE_FILE && !md5_equal(old->md5, cur->md5)) {
            // файл есть, но хеш изменился - содержимое изменено
            // для папок хеш = нули, поэтому проверяем только TYPE_FILE
            status = "CHANGED";
            color = "\033[33m";  // жёлтый
            stats[1]++;  // счётчик CHANGED
        } else {
            // файл есть и хеш совпадает - всё в порядке
            status = "OK";
            color = "\033[32m";  // зелёный
            stats[0]++;  // счётчик OK
        }
        printf("%-23.23s  [%s]  %s%-7s\033[0m\n", search, t, color, status);
        // %-23.23s - левое выравнивание, мин/макс 23 символа (обрезает длинные)
        // %s - строка типа "FILE" или "DIR "
        // %s - escape-код цвета
        // %-7s - статус с левым выравниванием, мин 7 символов
        // \033[0m - сброс цвета
    }

    // === ЦИКЛ 2: ищем NEW ===
    // проходим по всем записям из текущего скана

    for (size_t i = 0; i < vec_len(new_vec); i++) {
        file_rec *cur = vec_get(new_vec, i);
        if (cur->parent_id == 0)
            continue;  // пропускаем корневую запись

        build_path(new_vec, cur, new_path, sizeof(new_path));
        // строим путь в контексте new_vec

        if (has_prefix)
            snprintf(search, sizeof(search), "%s/%s", prefix, new_path);
            // для поиска в old_vec нужен полный путь: "utils/file.c"
            // потому что в old_vec пути хранятся относительно корня базы
        else
            strncpy(search, new_path, sizeof(search) - 1);

        if (!recursive && strchr(new_path, '/') != NULL)
            continue;
        // проверяем new_path (не search) - это путь относительно нашего скана

        if (find_by_path(old_vec, search) == NULL) {
            // в old_vec нет такой записи - файл новый, появился после сохранения базы
            const char *t = (cur->type == TYPE_FILE) ? "FILE" : "DIR ";
            printf("%-23.23s  [%s]  \033[36m%-7s\033[0m\n", new_path, t, "NEW");
            // \033[36m - голубой цвет
            stats[3]++;  // счётчик NEW
        }
    }
}
```

```c
#include "db.h"

int db_save(const char *path, vector_t *vec, int recursive) {
    if (path == NULL || vec == NULL)
        return 1;  // защита от NULL

    FILE *f = fopen(path, "wb");
    // "wb" = write binary (запись, бинарный режим)
    if (f == NULL) {
        fprintf(stderr, "Ошибка: не удалось открыть файл %s для записи\n", path);
        return 1;
    }

    db_header hdr;
    hdr.magic = IC_MAGIC;
    // IC_MAGIC = 0x41634463 = 'A'=0x41, 'c'=0x63, 'D'=0x44, 'c'=0x63 = "AcDc"
    hdr.record_count = (unsigned int)vec_len(vec);
    // сколько записей запишем - читается при загрузке чтобы знать сколько читать
    hdr.recursive = recursive;   // сохраняем флаг рекурсии
    hdr.saved_at = time(NULL);   // текущее время в секундах от эпохи Unix

    fwrite(&hdr, sizeof(db_header), 1, f);
    // записываем структуру заголовка как есть, байт за байтом
    // &hdr - адрес структуры, sizeof(db_header) - сколько байт, 1 - один элемент

    for (size_t i = 0; i < vec_len(vec); i++) {
        file_rec *rec = vec_get(vec, i);
        fwrite(rec, sizeof(file_rec), 1, f);
        // записываем каждый file_rec целиком (284 байта)
    }

    fclose(f);  // закрываем файл, данные записываются на диск
    return 0;   // успех
}

vector_t *db_load(const char *path, int *recursive_out, time_t *saved_at_out) {
    if (path == NULL)
        return NULL;

    FILE *f = fopen(path, "rb");
    // "rb" = read binary
    if (f == NULL) {
        fprintf(stderr, "Ошибка: не удалось открыть файл %s для чтения\n", path);
        return NULL;
    }

    db_header hdr;
    if (fread(&hdr, sizeof(db_header), 1, f) != 1) {
        // fread вернул 0 вместо 1 - файл пуст или меньше sizeof(db_header)
        fprintf(stderr, "Ошибка: не удалось прочитать заголовок базы\n");
        fclose(f);
        return NULL;
    }

    if (hdr.magic != IC_MAGIC) {
        // magic не совпадает - это не наш файл базы
        fprintf(stderr,
                "Ошибка: неверный формат файла (неправильный magic number)\n");
        fclose(f);
        return NULL;
    }

    // out-параметры: записываем значения если указатели не NULL
    if (recursive_out != NULL)
        *recursive_out = hdr.recursive;  // разыменовываем и записываем
    if (saved_at_out != NULL)
        *saved_at_out = hdr.saved_at;

    vector_t *vec = vec_create(hdr.record_count);
    // создаём вектор сразу нужного размера - не нужно расширять
    if (vec == NULL) {
        fclose(f);
        return NULL;
    }

    for (unsigned int i = 0; i < hdr.record_count; i++) {
        file_rec *rec = malloc(sizeof(file_rec));
        // каждый file_rec выделяем отдельно - vec_destroy освободит их по одному
        if (rec == NULL) {
            fprintf(stderr, "Ошибка выделения памяти\n");
            vec_destroy(vec);   // освобождаем всё что уже выделили
            fclose(f);
            return NULL;
        }
        if (fread(rec, sizeof(file_rec), 1, f) != 1) {
            // файл закончился раньше чем ожидалось - данные повреждены
            fprintf(stderr, "Ошибка: файл базы повреждён или обрезан\n");
            free(rec);          // этот rec ещё не в векторе - освобождаем сами
            vec_destroy(vec);   // остальные в векторе - vec_destroy освободит
            fclose(f);
            return NULL;
        }
        vec_push(vec, rec);
    }

    fclose(f);
    return vec;  // возвращаем заполненный вектор
}
```

## Полный листинг src/vector.c с разбором (ключевые функции)

```c
vector_t *vec_create(size_t ini_cap) {
    vector_t *temp = malloc(sizeof(vector_t));
    // выделяем память для самой структуры вектора
    if (temp == NULL)
        return NULL;

    if (ini_cap == 0)
        ini_cap = 1;  // минимальная ёмкость 1

    temp->cap = ini_cap;
    temp->len = 0;        // пустой вектор
    temp->owns_data = 1;  // будем освобождать элементы при уничтожении

    temp->data = malloc(temp->cap * sizeof(file_rec *));
    // массив указателей: cap ячеек, каждая sizeof(file_rec*) = 8 байт
    if (temp->data == NULL) {
        free(temp);
        return NULL;
    }

    size_t res_cap = (size_t)(temp->cap * 1.41);
    // резервный буфер на 41% больше основного
    if (res_cap <= temp->cap)
        res_cap = temp->cap + 1;  // на случай если cap=1 и 1*1.41=1 после приведения

    temp->res = malloc(res_cap * sizeof(file_rec *));
    // заранее выделяем резерв - при расширении не придётся ждать malloc
    if (temp->res == NULL) {
        free(temp->data);
        free(temp);
        return NULL;
    }

    return temp;
}

void vec_destroy(vector_t *vec) {
    if (vec == NULL)
        return;

    if (vec->owns_data) {
        // если вектор владеет данными - освобождаем каждый элемент
        for (size_t i = 0; i < vec->len; i++)
            free(*(vec->data + i));
            // *(vec->data + i) = vec->data[i] = указатель на i-й file_rec
            // free освобождает то на что указывает этот указатель
    }

    free(vec->data);  // освобождаем массив указателей
    free(vec->res);   // освобождаем резервный буфер
    free(vec);        // освобождаем саму структуру вектора
}

// Расширение вектора при заполнении (статическая, видна только в этом файле)
static int vec_reserve(vector_t *vec) {
    if (vec == NULL)
        return 1;

    // копируем указатели из data в res
    for (size_t i = 0; i < vec->len; i++)
        *(vec->res + i) = *(vec->data + i);
    // после этого и data[i] и res[i] указывают на одни и те же file_rec
    // это нормально - мы сейчас удалим data, поэтому два указателя не опасны

    free(vec->data);     // освобождаем старый основной буфер
    vec->data = vec->res; // резервный становится основным (мгновенно, без malloc!)

    // новая ёмкость = старая * 1.41
    size_t new_cap = (size_t)(vec->cap * 1.41);
    if (new_cap <= vec->cap)
        new_cap = vec->cap + 1;
    vec->cap = new_cap;

    // выделяем новый резервный буфер заранее для следующего расширения
    size_t res_cap = (size_t)(vec->cap * 1.41);
    if (res_cap <= vec->cap)
        res_cap = vec->cap + 1;

    vec->res = malloc(res_cap * sizeof(file_rec *));
    if (vec->res == NULL)
        return 1;  // ошибка - но данные уже в data, они сохранены

    return 0;  // успех
}

int vec_push(vector_t *vec, file_rec *rec) {
    if (vec == NULL || rec == NULL)
        return 1;

    if (vec->len == vec->cap) {
        // вектор полон - нужно расширить
        if (vec_reserve(vec))
            return 1;  // ошибка расширения
    }

    *(vec->data + vec->len) = rec;
    // то же самое что vec->data[vec->len] = rec
    // записываем УКАЗАТЕЛЬ на rec (не копируем сам rec)
    vec->len++;

    return 0;
}

file_rec *vec_get(vector_t *vec, size_t ind) {
    if (vec == NULL || ind >= vec->len)
        return NULL;  // защита от выхода за границы

    return *(vec->data + ind);
    // возвращаем указатель на file_rec по индексу
}
```

## MD5-хеш - как это работает (приближённо)

MD5 - это алгоритм хеширования: принимает данные любого размера, возвращает
16 байт (128 бит). Один и тот же файл всегда даёт один и тот же хеш.
Изменение хотя бы одного байта в файле полностью меняет хеш.

Реализация взята из https://github.com/Zunawe/md5-c
Используется функция `md5File(FILE *fp, uint8_t *result)`.

Приближённая схема работы MD5:
1. Файл читается блоками по 512 бит (64 байта).
2. Если последний блок неполный - дополняется специальным padding: бит 1, потом
   нули, в конце блока пишется длина исходного сообщения в битах.
3. Каждый блок обрабатывается через 64 раунда преобразований.
4. Каждый раунд применяет нелинейную функцию (смесь AND, OR, XOR, NOT),
   прибавляет константу, и делает побитовый сдвиг (rotate left).
5. 64 константы выбраны из таблицы floor(2^32 * abs(sin(i+1))) для i=0..63.
   Это даёт "случайные" константы без специального смысла.
6. После всех блоков получается 128-битный результат из 4 слов по 32 бита.

Почему хеш меняется при изменении файла: любое изменение входных данных
проходит через нелинейные операции и лавинообразно меняет весь результат
("лавинный эффект" / avalanche effect). Изменение 1 бита входа меняет
в среднем половину битов выходного хеша.

В программе хеш сравнивается побайтово в функции md5_equal().
Папки имеют хеш из нулей (memset(rec->md5, 0, 16)) - для папок
хеш не вычисляется, только для файлов.


## ANSI escape-коды - механика цветного вывода

Терминал в Linux понимает специальные последовательности байт внутри текста.
Когда терминал встречает `\033[` (байт 27, потом '['), он читает дальше
до буквы 'm' и интерпретирует как команду изменения цвета/стиля.

`\033` - восьмеричная запись числа 27 (0x1B), это символ ESC.

Таблица кодов цветов текста:
```
\033[31m  - красный    (MISSING)
\033[32m  - зелёный    (OK)
\033[33m  - жёлтый     (CHANGED)
\033[36m  - голубой    (NEW)
\033[0m   - сброс всех атрибутов (возврат к обычному цвету)
```

В коде:
```c
const char *color = "\033[32m";
printf("%s%-7s\033[0m\n", color, status);
```
Сначала печатаем управляющую последовательность включения цвета,
потом текст, потом `\033[0m` - отключаем цвет.
Если не отключить - весь последующий вывод тоже будет цветным.

Эти коды работают только в терминале. Если перенаправить вывод в файл
(`./integrctrl -c ... > out.txt`), в файле будут видны сырые байты `^[[32m`.

## Построение пути - подробный пример

Допустим сохранили папку `src` с флагом -r. Записи в базе:

```
id=1 name="src"      parent_id=0  type=DIR   (корень)
id=2 name="main.c"   parent_id=1  type=FILE
id=3 name="sub"      parent_id=1  type=DIR
id=4 name="helper.c" parent_id=3  type=FILE
```

Вызов build_path для id=4 ("helper.c"):
1. parent_id=3, не ноль. Ищем parent - id=3 ("sub").
2. parent_id id=3 равен 1. Ищем parent id=3 - это id=1 ("src"), parent_id=0.
   Значит parent_id родителя id=3 это 1, а 1 != 0, идём в рекурсию.
3. Вызываем build_path для id=3 ("sub"):
   - parent_id=1, ищем - id=1 ("src"), его parent_id=0.
   - parent_id родителя == 0 → копируем имя: buf = "sub". Возврат.
4. Возвращаемся в вызов для id=4. buf = "sub". len=3.
   buf[3]='/', buf[4]='\0', strncat дописывает "helper.c".
5. Итог: buf = "sub/helper.c".

В check_integrity функция find_by_path() вызывает build_path для каждой записи
и сравнивает результат с нужным путём. Это позволяет найти соответствие между
записью в базе и текущим файлом.
