# integrctrl - шпора для защиты

## Что делает программа

Утилита контроля целостности файлов через MD5-хеши.

- `-s` - сохранить хеши файлов в базу
- `-c` - проверить что изменилось
- `-r` - рекурсивный обход (вглубь папок)
- `-f <файл>` - путь к файлу базы

```
./integrctrl -s -r -f data/base.db src/
./integrctrl -c -r -f data/base.db
```

Выводит статус каждого объекта: OK / CHANGED / MISSING / NEW

---

## Структуры

**file_rec** - одна запись о файле или папке:
```
id, name[256], type (FILE/DIR), parent_id, md5[16]
```
Полный путь не хранится. Хранится только имя + ссылка на родителя (parent_id).
Корневая запись - parent_id=0.

**prog_opts** - параметры запуска:
```
mode ('s'/'c'), recursive (0/1), db_file, target_dir
```

**db_header** - заголовок бинарного файла базы:
```
magic (0x41634463 = "AcDc"), record_count, recursive, saved_at
```

**vector_t** - динамический массив указателей на file_rec:
```
data (массив указателей), res (резервный буфер), len, cap, owns_data
```

---

## Файлы и их роль

| Файл | Что делает |
|------|-----------|
| main.c | parse_args → run_save или run_check |
| integrctrl.c | parse_args, scan_directory, run_save, run_check |
| check.c | check_integrity - сравнивает old_vec и new_vec |
| db.c | db_save - пишет базу, db_load - читает базу |
| vector.c | динамический массив: push, pop, get, resize |
| md5.c | алгоритм MD5 (сторонняя реализация) |

---

## Главные функции

**scan_directory(dir_path, parent_id, next_id, vec, recursive)**
Обходит папку через opendir/readdir/closedir.
Для каждого файла считает MD5 через md5File(fp, rec->md5).
Для папок при recursive=1 вызывает сама себя (рекурсия).

**db_save(path, vec, recursive)**
Открывает файл "wb", пишет db_header, затем все file_rec подряд.
Бинарный формат: [заголовок][запись][запись]...

**db_load(path, recursive_out, saved_at_out)**
Читает заголовок, проверяет magic. Выделяет malloc для каждого file_rec,
читает fread, добавляет в вектор.

**check_integrity(old_vec, new_vec, prefix, stats[4], recursive)**
Цикл 1: по old_vec - ищет каждую запись в new_vec → OK / CHANGED / MISSING.
Цикл 2: по new_vec - ищет в old_vec, если нет → NEW.
stats: [0]=OK [1]=CHANGED [2]=MISSING [3]=NEW

**build_path(vec, rec, buf, size)**
Строит относительный путь, идя вверх по parent_id.
Рекурсивная функция. Пример: id=4 "helper.c" → "sub/helper.c"

## Бинарный файл базы

```
[db_header = 20 байт]
[file_rec = 284 байт] × N
```

file_rec: 4+256+4+4+16 = 284 байта.
db_header: 4+4+4+8 = 20 байт.

---

## MD5 - коротко

- MD5 обрабатывает файл блоками по 64 байта
- Через 64 раунда нелинейных операций (AND/OR/XOR + сдвиги) получает 16 байт результата
- Любое изменение файла полностью меняет хеш (лавинный эффект)
- В программе: md5File(fp, rec->md5), для папок хеш = 16 нулей

## Цветной вывод - механика

Терминал понимает ANSI escape-коды - специальные байты внутри строки:
```
\033[31m  - красный  (MISSING)
\033[32m  - зелёный  (OK)
\033[33m  - жёлтый   (CHANGED)
\033[36m  - голубой  (NEW)
\033[0m   - сброс цвета
```
`\033` = байт 27 (0x1B) = символ ESC.
Терминал видит ESC + '[' + цифры + 'm' и переключает цвет.
После текста обязательно печатаем `\033[0m` - иначе цвет не сбросится.

В коде:
```c
printf("%-23.23s  [%s]  %s%-7s\033[0m\n", search, t, color, status);
```

## vec_reserve - интересная деталь

Вектор хранит два буфера: data (основной) и res (резервный).
Когда data заполнен:
1. Копируем указатели data → res
2. free(data)
3. data = res  (без нового malloc!)
4. Выделяем новый res побольше

Резервный буфер уже выделен заранее - расширение происходит без ожидания malloc.

---

## Почему полный путь не хранится

Хранить только name + parent_id - это дерево. Преимущества:
- Меньше места (нет повторяющихся частей пути)
- Переименование папки = изменение одной записи, а не всех дочерних

Недостаток: чтобы получить полный путь нужна рекурсивная функция build_path.

---

## Флаг -r в -c не наследуется из базы

```c
int recursive = opts->recursive;  // только из аргументов командной строки
```

db_recursive читается из файла, но НЕ используется для сканирования.
Если хочешь рекурсивную проверку - нужно явно указать -r.
Это сделано чтобы проверить только верхний уровень папки при необходимости.

---

## Prefix - фильтрация по подпапке

```
./integrctrl -c -f data/base.db src/utils
```

Если база записана для всей папки `src`, а проверяем только `src/utils`,
программа вычисляет prefix="utils" и показывает только те записи,
чей путь начинается с "utils/".
